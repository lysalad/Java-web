<인터넷 주소창(URL)의 구성>
http: -> 프로토콜을 말한다.
// -> 그냥 나누는 기능... 그냥 // 두개... 그냥 just
그 뒤의 주소는 도메인 이름으로 '컴퓨터 하나'를 말한다. DNS라고도 한다.
우리는 URL을 이용해 우리도 모르는 사이에 DNS에 들렀다가 온다.

접속과정은 다음과 같다.
URL을 통해 해당하는 IP를 가진 컴퓨터로 내 컴퓨터가 접속한다.
이 연결동안 내 켬퓨터와 IP를 가진 컴퓨터가 서로 소통한다.
우리는 요청을 보내기 때문에 클라이언트, 사용자이다.
사용자가 연결을 통해 서비를 해달라고 요청을 보내면, 받는 컴퓨터가 서버이다.
서버가 응답을 한 뒤에는 일단 연결이 끊어지는데,

왜...? 그런지는...? 설명을 잘 못들었따 ㅎㅎ 머쓱!

<상용 서버인 apache TomCat을 써보자!>
apache TomCat을 쓰는 이유는? 무료라서.. ㅎ...
apache TomCat은 자바 어플리케이션이 동작할 수 있는 서버를 준다.
일반적인 웹서버와는 조금 다르다고 한다...

나의 apache TomCat을 보려면 기본적인 주소를 치면 된다.
http://localhost:8080 이다.
apache TomCat을 압축 풀기 한 곳에서 
webapps로 들어가 그 안에 html 파일들을 만든다.

주소창에 해당 html파일이 있는 경로를 적어주면~
나만의 웹페이지 완성!

이때는 localhost:8080 대신에
cmd에서 ipconfig로 알아낸 나의 IP 주소를 적어주면 된다!
